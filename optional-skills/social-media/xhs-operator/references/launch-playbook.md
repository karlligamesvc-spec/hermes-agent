# 小红书起号操作手册

这份参考手册把小红书起号经验提炼成可复用、合规优先的操作流程。发布时间、推荐池、标签权重、薯条投放、蒲公英报备等说法都按经验假设处理,不视为官方或实时规则。

> **对标系统和选题库不在这份手册里。** 它们是本技能的核心真数据工作流,活在 `SKILL.md` 的第 3 步(对标拆解)和第 4 步(选题库)——那两节明确指示调用 `social_search` / `social_posts` / `creator_top_posts` / `social_content` / `social_comments`(`platform=xiaohongshu`)拉真实笔记数据,再抽模式。这里只放 prompt-only 的模板和检查表。

## 1. 需求采集表

用户要求起号计划、账号审计、选题库或笔记系统时使用。

| 字段 | 询问或推断 |
| --- | --- |
| 目标 | IP/人设信任、种草带货、品牌门店、引流转化、电商店铺或测试 |
| 赛道 | 类目、细分类目、首月最窄切入口 |
| 读者 | 谁有痛点、好奇心、消费能力或收藏动机 |
| 变现 | 账号最终要卖什么、验证什么或沉淀什么(蒲公英/佣金/私域/门店/电商) |
| 证明 | 实拍图/视频、案例、资质、经验、对比图、用户问题、研究笔记、故事素材 |
| 产能 | 每周笔记数、拍摄出图能力、封面制作、编辑支持、首轮冲刺周期 |
| 阶段 | 新号、沉寂号、内容不稳定、低曝光、限流风险、活跃中或放大期 |
| 约束 | 合规类别(医美/医疗/金融/减肥/教育)、广告法边界、版权、隐私、图片权利、品牌语气、导流规则 |

## 2. 起号模式判断

| 模式 | 适用情况 | 优先优化 | 注意事项 |
| --- | --- | --- | --- |
| IP/人设信任号 | 用户想做长期品牌、服务、产品或读者信任 | 清晰承诺、人设一致、稳定赛道、专业度、留存、转化路径 | 反馈较慢,避免追逐无关热点 |
| 种草/带货号 | 用户想测种草收益或低成本内容题材 | 细分品类、真实体验、生产系统、原创度、版权安全、复盘循环 | 波动大,不能承诺收益或上热门 |
| 修复/限流号 | 账号有旧内容、低曝光或定位漂移 | 诊断、清理、垂直冲刺、信任重建 | 先判断是否有违规导流、搬运或错误受众信号 |
| 放大型账号 | 已经有部分有效选题 | 模式扩展、主题簇、系列化、封面模板复用、转化 | 胜出模式没稳定前不要过度扩张 |

默认优先做 IP/人设信任系统。只有当用户明确接受风险和有限上限时,才设计纯流量/带货实验号。

## 3. 主页地基

先产出定位句:

`我帮助[目标读者],用[方法/证明/内容承诺]解决[具体痛点],让他们获得[理想结果]。`

把定位句转成:

- 头像:清晰、可识别、符合领域气质(人设号优先真人/固定形象)。
- 昵称:一个领域关键词,加一个有记忆点的人设或IP名。
- 简介:一句话写清身份、价值和更新承诺,自然嵌入可搜关键词。
- 背景图:强化人设/价值主张,可含一句 slogan。
- 置顶笔记方向:代表作/合集/自我介绍,承接主页第一印象。
- 主页引导话术:简介/背景图里的合规下一步动作(如"点关注看 XX 系列")。
- 内容支柱:3到5个支柱,分别承接搜索、种草、干货、故事和转化。
- 关键词宇宙:品类词、痛点词、场景词、人群词、对标邻近词(既做标签也做可搜标题词)。

### 账号准备度检查表

| 模块 | 检查点 |
| --- | --- |
| 主页 | 头像、昵称、简介、背景图、置顶方向、主页引导都指向同一个读者和价值 |
| 信号 | 首轮内容保持垂直,核心关键词自然出现在标题/正文/标签中 |
| 素材 | 起号前至少准备10到20份源素材:实拍图/视频、案例、笔记、问题、对比、故事 |
| 对标 | 已用 social 工具真实拉过 5-10 个对标账号、20-50 篇对标笔记(见 SKILL.md 第 3 步) |
| 封面 | 已定 2-3 套封面模板方向(大字/实拍/对比),每条笔记出 3 版选 1 |
| 发布 | 有笔记简报、30天内容日历、复盘表和风险检查表 |
| 合规 | 不做虚假互动、刷量、私信自动化、隐藏联系方式、广告法极限词、盗图盗视频 |

## 4. 标题钩子模式

标题钩子用于设计备选,不用于夸大事实。在 SKILL.md 第 3 步从真实对标标题里归纳时,可对照这张表命名模式。

| 模式 | 用法 |
| --- | --- |
| 数字法 | 让价值具体,例如"月薪5千也能做到的7个习惯" |
| 痛点前置法 | 开头直戳痛点,例如"皮肤黄一白遮三丑?" |
| 反差法 | 制造张力,例如"我以为是智商税,用完真香" |
| 疑问法 | 直接说出读者心里的问题 |
| 身份认同法 | 圈定人群,例如"打工人/学生党/新手妈妈必看" |
| 好奇法 | 留一个具体的信息缺口 |
| emoji 断句法 | 用 emoji 分隔关键信息,提高扫读点击 |
| 避坑法 | "别再踩这些坑""XX 千万别买",制造损失规避 |
| 高赞评论角度 | 从真实评论(`social_comments`)重建角度,不复制原文 |

每条笔记先写8到12个标题,再按读者意图、具体程度、可信度和广告法风险选择。**极限词(最/第一/国家级/永久/根治等)一律不用。**

## 5. 封面模式清单

小红书封面是第一入口。在 SKILL.md 第 3 步从真实爆款封面归纳时,可对照这张表命名模式;第 5 步出 3 版封面时从中选不同方向。

| 封面模式 | 用法 | 适合 |
| --- | --- | --- |
| 大字纯色底 | 主标题超大字 + 纯色/浅底,信息一眼看清 | 干货、清单、观点 |
| 人物实拍叠字 | 真人出镜 + 左上/顶部标题字 | 人设号、穿搭、美妆 |
| 前后对比 | 左右/上下 before-after + 结果字 | 变美、改造、测评 |
| 清单九宫格 | 多图拼合 + 序号 | 合集、好物、步骤 |
| 实拍产品特写 | 产品清晰特写 + 卖点字 | 种草、开箱 |
| 手写/便签感 | 手写字体/便签背景,亲切真实 | 笔记、经验分享 |
| 表格截图 | 对比表/清单表直接当封面 | 攻略、对比 |

**封面文案原则:** 主标题≤两行、大而可读;副信息补一句;不用极限词;不搬对标封面,由生图能力生成 3 版或用户自有素材制作。

## 6. 笔记简报模板

写文案前先填这张表(SKILL.md 第 5 步的完整字段)。

| 模块 | 内容 |
| --- | --- |
| 笔记任务 | 搜索 / 种草 / 干货 / 故事 / 情绪 / 避坑 / 转化 |
| 目标读者 | 谁最应该点开这条笔记 |
| 读者状态 | 痛点、好奇、误解、恐惧、渴望、反对意见 |
| 关键词 | 1个主关键词,加2到4个相关词 |
| 标签组 | 3到6个话题标签(来自真实数据,SKILL.md 第 3/4 步) |
| 标题组 | 8到12个标题,并标出推荐标题 |
| 封面文案 | 主标题字(≤两行)+ 副信息 + 封面模式方向(出 3 版) |
| 开头钩子 | 前1到2行:痛点、结果、场景或反差 |
| 正文结构 | 图文:3到6点清单/步骤;视频:分镜脚本(镜头+口播+字幕) |
| 证明材料 | 用户自有实拍、案例、对比图、经验、来源或过程 |
| 风格 | 口语化、短句、emoji 断句,必要时使用"你/我" |
| 行动引导 | 与笔记任务匹配的一个合规动作(关注/收藏/评论/主页看更多) |
| 风险检查 | 广告法极限词、版权(图/视频)、隐私、医美/医疗/金融/减肥、导流合规、平台规则 |

### 闺蜜脑改写

草稿写完后,把它当作发给闺蜜的一段真心分享:

- 把端着的表达改成能说出口的话。
- 拆短句子,用 emoji 分隔关键信息。
- 加入具体场景、例子和利害关系。
- 删除空泛总结段。
- 一条笔记只保留一个核心收藏理由。

## 7. 提示词模板

以下模板只能作为起点,必须按用户赛道、素材、证明和语气大幅定制。**注意:模板里的"对标"步骤应替换为 SKILL.md 第 3 步的真实工具调用,而不是让模型凭空想象对标账号。**

### 图文笔记提示词

```text
你正在为[账号定位]起草一条小红书图文笔记。

目标读者:[读者]
笔记任务:[搜索/种草/干货/故事/情绪/避坑/转化]
核心痛点或好奇心:[痛点]
主关键词:[关键词]
话题标签:[来自真实数据的 3-6 个标签]
用户自有素材:[实拍图、案例、对比、故事、研究资料]
指定角度:[角度]
风险边界:不用广告法极限词,不复制文字,不盗图盗视频,不编造数据,不设计私信自动化或隐藏联系方式。

先输出原创笔记简报:
1. 10个标题选项,并标出最好的3个。
2. 封面文案:主标题字(≤两行)+ 副信息 + 3 个封面方向。
3. 开头钩子选项(前1-2行)。
4. 3到6点正文清单/步骤大纲。
5. 每点需要的证明材料(优先用户实拍)。
6. 合规主页引导 / 收藏引导。
7. 广告法和原创度风险自查。

然后用中文写正文,要求口语化、短段落、emoji 断句、例子具体。
```

### 视频笔记提示词

```text
为[账号定位]创作一条小红书视频笔记脚本。

目标读者:[读者]
主题:[主题]
时长目标:[15-60 秒]
主关键词与标签:[关键词 + 3-6 标签(来自真实数据)]

输出:
1. 首帧封面文案(标题字≤两行 + 方向,出 3 版)。
2. 前 3 秒钩子(留住划走的人)。
3. 分镜脚本表(镜头 / 口播 / 屏幕字幕 / 时长)。
4. 结尾行动引导(合规:关注/收藏/主页)。
5. 广告法与版权自查。

口播口语化,字幕短促,前 3 秒必须给出结果或反差。不用极限词,不搬运他人素材。
```

## 8. 首个30天起号日历

| 阶段 | 天数 | 目标 | 工作 |
| --- | --- | --- | --- |
| 地基 | 1-3 | 建立账号信号和读者承诺 | 定位句、主页六件、封面模板方向、3到5个内容支柱 |
| 对标 | 4-7 | 建立模式库 | 用 social 工具真实拉 5-10 个账号、20-50 篇笔记,拆标题/封面/评论/标签(SKILL.md 第 3 步) |
| 草稿储备 | 5-10 | 避免临时赶稿 | 准备10到15条简报或草稿 + 封面各出 3 版选 1 |
| 首轮冲刺 | 8-20 | 测试垂直信号 | 在一个赛道内稳定发布,避免突然跨类目;固定封面模板 |
| 模式扩展 | 21-27 | 复用早期有效模式 | 改写有效角度,做系列,优化标题/封面/标签 |
| 复盘 | 28-30 | 确定下轮重点 | 排名选题、标题、封面、标签、行动引导、产能瓶颈和风险;淘汰弱选题 |

源经验提到日更和特定时段发布,这只能作为实验假设,不是保证。可持续质量优先于强行日更。

## 9. 指标诊断

按失败环节诊断。**精确曝光/点击率/完播/涨粉/收藏率数据需用户从小红书创作者后台截图提供,外部工具采不到。**

| 现象 | 可能原因 | 修复动作 |
| --- | --- | --- |
| 曝光低 | 账号信号不清、赛道漂移、可能限流(导流/搬运/违规)、首轮内容弱、标签不精准 | 收紧主页,保持垂直,检查违规导流,优化可搜关键词与标签 |
| 有曝光但点击低 | 封面弱、标题承诺弱、选题泛、读者不匹配 | 重做封面(3 版选 1),重写标题组,用更具体的痛点/反差 |
| 点开后完读/完播低 | 开头慢、语言端着、没张力、抽象太多 | 做闺蜜脑改写,前 1-2 行给结果/反差,加具体例子 |
| 阅读有但收藏低 | 不够实用,缺清单/模板/攻略/干货密度 | 增加可带走资产、清单、步骤、对比表 |
| 阅读有但评论低 | 没有明确问题,没有身份或决策张力 | 结尾问一个和读者处境有关的具体问题 |
| 互动有但关注低 | 内容有用,但账号/人设承诺不清 | 强化简介、系列承诺、置顶笔记和内容一致性 |
| 关注有但转化弱 | 行动引导不清或太早硬广 | 用合规主页引导/官方留资功能,别用私信自动化 |

类别内相对基准比通用阈值更重要。**小红书当前无公开热门榜数据源(hc-426 上线后接入),热点判断暂靠关键词搜近期高互动笔记近似。**

## 10. 风险规则与合规替代

把高风险技巧改成安全做法。

| 高风险诉求 | 处理方式 |
| --- | --- |
| "能不能让朋友帮忙点赞收藏冲数据?" | 提醒人为互动会污染受众信号并有限流风险;建议私下收集质量反馈,而不是制造平台行为 |
| "能不能买赞/刷量/刷收藏?" | 拒绝;转向封面、标题、选题、收藏率和复盘机制优化 |
| "能不能挂第三方工具自动私信/关键词群发引流?" | **拒绝(红线)**;私信自动化违规且伤号,改用合规主页引导或小红书官方留资功能 |
| "怎么搬运/洗稿爆款笔记还不被发现?" | 拒绝洗稿和规避检测;只抽结构,用原创实拍和自有证明重建 |
| "封面/图片能不能直接从对标笔记拿?" | 拒绝盗图盗视频;用生图能力生成或合法授权/自有实拍素材 |
| "标题能不能写'全网最便宜/第一/永久有效'?" | 拒绝广告法极限词与功效承诺;改成可核验的具体表达 |
| "低曝光要不要注销重来?" | 先诊断违规导流、定位漂移和质量问题;只有有明确战略理由时才重开 |
| "怎么在笔记里藏微信/外链导流?" | 拒绝绕审核;改用合规主页引导、官方私信菜单或商业功能 |

建议话术:

- "我不能帮你设计私信自动化或规避审核的方法,但可以把它改成合规的内容和主页引导系统。"
- "对标只学结构,不搬原文和封面。我们用你的实拍素材重新做。"
- "标题不用极限词,涉及医美/减肥/金融等敏感内容时,先核验当前小红书规则和广告法。"
