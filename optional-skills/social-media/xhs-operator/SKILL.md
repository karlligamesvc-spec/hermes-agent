# 小红书运营专家 (xhs-operator)

## Overview

把自己当成小红书起号策略师,而不是捷径复述器。根据用户的赛道、变现路径、素材资产、风险约束和账号阶段,搭建一套能执行的起号系统:定位、主页地基、对标体系、选题库、笔记简报(文案+封面)、内容日历、风险控制和复盘循环。

**这套技能与纯 prompt 起号包的核心区别:对标拆解和选题库不靠模型编数据,而是调用平台社媒数据工具拉取真实笔记。** 纯 prompt 包的死穴是"对标账号""爆款封面""热门标签"全靠模型脑补;这里的对标模式来自真实搜索结果。凡是外部采不到的数据(尤其小红书后台的曝光/点击/涨粉/薯条投放),诚实说明并让用户截图喂进来,绝不编造。

小红书是**封面驱动 + 搜索驱动**的双入口平台:封面(图/视频首帧+标题字)决定点击,标题和标签决定被搜到和进推荐。把发布时间、推荐池、标签权重、薯条投放效果、蒲公英/报备规则等说法都视为经验假设。凡涉及当前平台规则、商业合作报备、违规处罚、导流政策等变化较快的信息,在给最终运营建议前先核验最新规则。**不承诺涨粉数、爆款结果、上热门或进入流量池。**

## When to Use

- 用户要做小红书起号计划、新号冷启动、沉寂号重启,或问账号定位/主页地基。
- 用户要拆解对标账号、分析爆款标题/封面模式、找热门标签、建选题库,或写笔记简报(文案+封面指令)。
- 用户要排 30 天内容日历,或做小红书周复盘、诊断曝光/点击/互动/涨粉环节。

**Don't use for:** 公众号/抖音/视频号起号(用对应平台技能);单纯写一条不涉及起号系统的笔记(直接写即可);刷量/互赞/养号矩阵/私信自动化群发等诉求(见"风险边界",转为合规替代)。

## 首轮信息清单

只补齐真正缺失的信息。用户已给足上下文就直接产出方案。

- **账号目标**:IP/人设信任号、种草带货号、品牌/门店号、引流转化号(引私域/门店/电商),或混合型。
- **赛道与目标读者**:谁会点开、点赞、收藏、关注或转化(小红书主力女性/生活方式/消费决策场景)。
- **变现路径**:蒲公英品牌合作、带货佣金、私域引流、门店到店、电商店铺、付费咨询,或暂不变现。
- **当前阶段**:全新号、沉寂号、内容漂移号、低曝光号、限流/违规风险号、已活跃号。
- **现有资产**:专业经验、成品图/视频素材、案例、对比图、用户问题、产品实拍、版权授权情况。
- **产能**:每周笔记数、拍摄/出图能力、写作速度、封面制作支持、起号周期。
- **约束**:敏感行业(医美/医疗/金融/教育/减肥)、广告法边界、隐私、版权、平台导流规则、品牌语气。

完成判据:七类信息都有值(实测或用户明确给出),或已明确标注"用户未提供、按默认假设推进"。

## 工作流程

### 1. 判断起号模式

- **IP/人设信任号**:优先做清晰定位、人设一致性、内容深度、可信证明、留存和转化路径。
- **种草/带货实验号**:优先做细分品类、选题生产系统、真实体验、版权安全(不盗图盗视频)和退出标准。
- **沉寂/限流号**:审计定位漂移、低质内容、违规导流、搬运风险、限流信号,以及修复或重开的必要性。
- **放大型账号**:已有部分有效选题,做模式扩展、主题簇、系列化、封面模板复用、转化;胜出模式没稳定前别过度扩张。

默认优先 IP/人设信任系统。只有用户明确接受风险和有限上限时,才设计纯流量/带货实验号。

### 2. 写出定位句并落成主页地基

定位句模板:`我帮助[目标读者],用[方法/证明/内容承诺]解决[具体痛点],让他们获得[理想结果]。`

把定位句转成主页六件:头像气质、昵称(一个领域关键词 + 一个记忆点人设/IP名)、一句话简介(身份+价值+可搜关键词)、背景图、置顶笔记方向、以及**主页引导话术**(简介里的合规下一步动作)。让主页所有模块指向同一个读者和价值;起号首轮保持垂直,账号信号未稳前避免跨赛道。

**转化路径只做内容策略层(硬边界,见"风险边界"):** 主页引导写在简介/背景图/置顶笔记里(如"点关注看 XX 系列""合作看主页");评论区口径用统一话术回应高频问题。**私信自动化(自动回复群发、关键词批量私信、第三方挂机私信工具)= 红线,本技能不设计、不提供操作说明。** 需要留资时改用小红书官方功能(如企业号私信菜单、留资组件)或合规主页引导。

完成判据:主页六件(头像/昵称/简介/背景图/置顶方向/主页引导话术)都对齐同一读者,核心可搜关键词已列出,转化路径只落在内容层且未触私信自动化红线。

### 3. 对标拆解(真数据工作流 — 本技能核心)

**不要凭记忆列对标账号或编封面/标题。用平台社媒数据工具拉真实笔记,再抽取模式。** 全程 `platform="xiaohongshu"`。

**采集步骤:**

1. **按赛道关键词搜真实笔记** — 调 `social_search`:
   ```json
   { "name": "social_search", "arguments": { "platform": "xiaohongshu", "query": "<赛道/痛点关键词>", "count": 20 } }
   ```
   用 3-5 组关键词(品类词 / 痛点词 / 场景词 / 人群词)各搜一轮,汇总候选笔记与所属账号。

2. **锁定对标账号,拉其近期笔记 + 高互动笔记** — 对搜索结果里反复出现、且互动明显的账号:
   ```json
   { "name": "social_posts", "arguments": { "platform": "xiaohongshu", "user_id": "<对标号 id>", "count": 20 } }
   ```
   要按互动量筛爆款时,用 `creator_top_posts`(小红书是它的支持平台之一,按点赞/收藏阈值枚举打分):
   ```json
   { "name": "creator_top_posts", "arguments": { "platform": "xiaohongshu", "user_id": "<对标号 id>", "min_likes": 10000, "sort_by": "likes", "top": 20 } }
   ```
   可选 `social_profile`(`platform=xiaohongshu, user_id=<id>`)核对账号定位与主页承诺。

3. **抽取单篇笔记结构** — 对要深拆的爆款笔记,调 `social_content` 取正文:
   ```json
   { "name": "social_content", "arguments": { "platform": "xiaohongshu", "url": "<笔记链接>" } }
   ```
   (有 note_id 时传 `item_id`。)

4. **读评论区拿真实读者语言 + 真实标签** — 调 `social_comments`,从真实评论重建角度、痛点、反对意见和收藏动机(不复制原文):
   ```json
   { "name": "social_comments", "arguments": { "platform": "xiaohongshu", "item_id": "<note_id>" } }
   ```

**从真实数据抽取(填对标记录表):**

| 字段 | 从工具结果里抽 |
| --- | --- |
| 账号 | 昵称、定位、赛道、读者、可见点赞/收藏/关注信号(来自 search/profile/posts/top_posts) |
| 笔记 | 链接、标题、日期、主题、形式(图文/视频)(来自 search/posts) |
| 封面模式 | 大字标题 / 人物出镜 / 前后对比 / 清单九宫格 / 实拍产品 / 手写感 / 表格截图 —— 从爆款封面归纳(读 content 里的封面/图片信息与标题) |
| 标题钩子 | 读者为什么点开(读标题,来自 search/posts;数字 / 痛点前置 / 反差 / 疑问 / 身份认同 / emoji 断句) |
| 结构 | 干货清单 / 教程步骤 / 种草测评 / 个人故事 / 避坑 / 对比 / 合集(来自 content 正文) |
| 情绪触发 | 好奇 / 身份 / 焦虑 / 松弛 / 省钱 / 变美变好(来自 content + comments) |
| 标签 | 笔记实际带的话题标签 + 评论高频词(来自 content 的 tags/正文 + comments,作为热门标签候选) |
| 证明 | 实拍 / 数据 / 对比图 / 资质 / 用户反馈(来自 content) |
| 行动引导 | 关注 / 点赞收藏 / 评论 / 主页看更多 / 搜索关键词(来自 content 结尾) |
| 重建方向 | 如何用用户自己的素材改写成新笔记 |

**诚实边界(必须遵守):**

- **小红书后台数据(精确曝光、点击率、涨粉、收藏率、薯条/聚光投放效果、蒲公英报价)外部工具采不到。** 工具能给的是公开可见的标题、正文、封面信息、评论、以及平台暴露的公开互动信号(点赞/收藏/评论数),不是创作者后台面板。需要精确曝光/点击/涨粉时,明说"这项外部采不到,请从小红书创作者后台截图给我"。
- 工具返回 `vendor_unavailable` / 平台能力暂不可用时,**显式告知"小红书数据能力暂不可用",不要静默、不要本地自装替代抓取器**,并说明可改为用户手动贴笔记链接/截图。
- 只抽结构,不搬原文、不搬图片/封面。对标账号的标签/话题只作研究线索,不承诺带来推荐流量或上热门。

完成判据:至少用 `social_search` 真实搜过并列出候选账号/笔记;对标记录表每行都能追溯到某次工具调用的返回,而非模型脑补;后台类数据已明确标注"需用户截图"。

### 4. 选题库(真数据工作流 — 本技能核心)

选题库是**字段表**,种子来自第 3 步的真实搜索/对标结果,不是凭空列题。热点选题当前用 `social_search`(以时效性/季节性关键词搜近期高互动笔记)。`<!-- 小红书热门榜数据源上线后接入(hc-426):届时用热榜话题批量喂 social_search 的 query,做热点选题;当前无热门榜端点,只能靠关键词搜近期高互动笔记近似。 -->`

对每条选题填:

| 字段 | 内容 |
| --- | --- |
| 选题 | 一句话说清这条笔记要解决/回答什么 |
| 读者痛点 | 谁的什么痛点/好奇/误解(优先来自 `social_comments` 的真实读者语言) |
| 关键词 | 1 个主关键词 + 2-4 个相关词(来自 `social_search` 命中的高频词) |
| 标签建议 | 3-6 个话题标签(来自第 3 步真实笔记带的标签 + 评论高频词,标注来源;别硬编热门标签) |
| 角度 | 搜索 / 种草 / 干货 / 故事 / 情绪 / 避坑 / 转化 中的一种 |
| 标题+封面方向 | 从对标真实爆款归纳出的标题钩子 + 封面模式(如"数字大字封面""前后对比封面"),不是抄原封面 |
| 素材源 | 用户自有实拍/案例/经验,或某条对标笔记的结构参考(注明来源链接) |
| 风险检查 | 广告法极限词 / 版权(图/视频) / 隐私 / 医美医疗金融减肥 / 违规导流 / 平台规则风险 |

产出默认 10 条起(用户要更多再加)。每条的"痛点""关键词""标签建议"尽量能指回一次真实工具调用;实在只有经验判断的,标注"(经验假设,未经数据核验)"。

完成判据:选题库 ≥10 条,字段齐全;至少半数选题的痛点/关键词/标签可追溯到 `social_search` 或 `social_comments` 的真实返回;热点行留有小红书热门榜(hc-426)接入注释。

### 5. 笔记简报 → 文案 + 封面(两段式)

**先写简报,再出文案和封面指令。** 简报字段(完整模板见 references):笔记任务、目标读者、读者状态、关键词、标签组、标题组(8-12 个,标出推荐)、封面文案(主标题字≤两行 + 副信息)、开头钩子(前 1-2 行)、正文结构(图文清单/视频脚本分镜,3-6 点)、证明材料、行动引导、风险检查(广告法极限词自查)。

- 小红书**封面 + 标题负责点击**,正文负责完读、点赞、收藏、关注和转化。
- 文案写完做"闺蜜脑改写":口语化、短句、加具体场景和 emoji 断句、删空泛总结、一条笔记只保留一个核心收藏理由。

**封面(生成 3 版选 1):** 简报确认后,调用运行时的**图片生成能力生成封面(系统按档位自动选路由)**——带字封面走默认档位(适合带标题字的封面);另有免费封面线(平台底图 + 排版叠字)在系统档位内可选。**技能只描述"生成一版带标题字的封面",不写死用哪个 provider/模型**——路由由运行时按档位决定。用一句话 prompt 描述封面:主标题文字 + 画面主体 + 风格 + 排版(如"左上大字标题")。**出 3 版**不同封面方向(如:大字纯色底 / 人物实拍叠字 / 前后对比),让用户选 1 版落地。

**若当前运行时未提供图片生成能力(工具不存在或返回不可用):跳过封面生成并说明"当前环境无生图能力,封面请手动制作(给排版建议即可)",不要编造一张图或谎称已生成。**

完成判据:先出简报后出文案;标题组 ≥8;标签组来自真实数据;做过闺蜜脑改写;封面要么真生成 3 版选 1、要么诚实跳过并给排版建议;广告法极限词自查过。

### 6. 内容日历(30 天)与周复盘

**30 天内容日历落表**(模板见 references):按周排选题、笔记类型(图文/视频)、封面方向、发布节奏。优先可持续稳定,不为日更牺牲质量;新号正式发布前尽量备一批简报/草稿。

**周复盘 = 数据回采对比 + 淘汰弱选题。** 明确区分两类数据:

| 数据来源 | 能拿到什么 | 怎么拿 |
| --- | --- | --- |
| **外部可采(对标动态)** | 对标账号近期发了什么、标题/封面结构变化、评论区新痛点、哪些标签在热 | 重跑第 3 步的 `social_search` / `social_posts` / `creator_top_posts` / `social_comments`,和上周对比 |
| **用户截图喂入(自己后台)** | 本号真实曝光、点击率、涨粉、收藏率、单条数据、薯条/聚光投放效果 | **外部工具采不到,让用户从小红书创作者后台/投放后台截图**,读图后录入 |

复盘输出三段:

1. **有效模式** — 哪些选题支柱/标题钩子/封面模式/标签反复带来正向信号(结合两类数据)。
2. **失败假设(淘汰弱选题)** — 哪些选题/封面/标题假设被数据证伪,下轮日历里淘汰或改造。
3. **下周实验** — 基于以上,列 2-3 个可检验的下周动作(新选题角度/新封面模板/新标题实验/新标签组合)。

**明示数据边界:** 复盘开头写清"本复盘里 X/Y/Z(曝光/点击/涨粉)来自你提供的后台截图;对标动态来自公开数据工具;二者精度不同"。缺后台截图时,只做对标侧复盘,并说明自号侧数据待补。**小红书当前无公开热门榜数据源(hc-426 上线后接入),热点判断暂靠关键词搜近期高互动笔记近似,已在复盘里注明。**

完成判据:30 天日历落表;复盘区分了两类数据来源;三段(有效/失败淘汰/实验)齐全;明确标注了哪些自号数据缺失待用户截图 + 热门榜数据源待接入。

## 输出格式

除非用户要完整战略,默认给最小但有用的产物:

- **起号计划**:定位、主页文案、内容支柱、30天日历、指标、复盘节奏。
- **账号审计**:问题、证据、风险等级、优先修复项、未来 7 个动作。
- **对标拆解**:账号/笔记、标题+封面结构、开头、正文框架、标签、证明、行动引导、改写方向(每行可追溯工具调用)。
- **选题库**:字段表 ≥10 条(选题/痛点/关键词/标签建议/角度/标题+封面方向/素材源/风险检查)。
- **笔记简报**:标题组、标签组、封面文案、开头钩子、大纲、证明、行动引导、原创度+广告法检查。
- **内容日历**:30 天落表(周/选题/类型/封面方向/节奏)。
- **周复盘**:有效模式、失败假设(淘汰弱选题)、下周实验(数据来源两分)。

## 文案模型选择建议

写标题组/笔记文案/封面文案这类文本时的模型偏好(与封面生图的档位路由是两回事,别混):

- **默认用 `deepseek-v4-pro`**——综合网感、信息密度、诚实度、可发布度最均衡,不需要特意换。
- **诚实度敏感选题(医美、医疗、减肥、金融、数据类)可换 `qwen3.7-max`**——它诚实不夸大一维得分最高,更不容易踩广告法极限词/夸张承诺(仍要过一遍"风险边界"的广告法极限词检查)。
- **别用 `kimi-k2.6` 生成文案**——可直接发布度明显偏低,产出往往还要大改。
- `glm-5.2` 在盲评里小红书平台分意外偏高,但样本量小、还不足以下结论;想试可作对照,别据此改默认。

依据:2026-07-06 五模型盲评(30 题×5 模型,去掉模型名盲评判卷,维度=网感/信息密度/诚实不夸大/可发布度);明细见 `scripts/eval/out/report.md`。结论会随后续评测更新,以最新报告为准。

## 风险边界

不提供刷量、互点互赞、养号矩阵滥用、人为干预流量、绕过审核、**私信自动化群发 / 关键词批量私信 / 第三方挂机私信工具**、隐藏违规联系方式、搬运/洗稿规避检测、绕过原创识别等操作说明。源材料含这类做法时,转成风险提示并给合规替代方案。

**转化路径硬边界:只做内容策略层(主页引导话术 + 评论区统一口径)。** 私信自动化一律红线;需要留资/导流时改用小红书官方功能或合规主页引导,不设计任何自动化私信/群发/挂机方案。

医美、医疗、金融、投资、减肥、教育承诺、未成年人、法律、收入承诺、公共议题等敏感类别,写作前先加一轮**广告法极限词 + 宣传边界检查**(禁"最/第一/国家级/永久/根治"等极限词与功效承诺)。

图片、视频、笔记、截图等有版权风险的材料只作研究输入。最终稿基于原创实拍、原创分析、合理引用、用户自有素材或合法授权素材重建;**封面不搬对标图,由生图能力生成或用户自有素材制作。**

**数据诚实铁律:** 拿不到的数据(尤其小红书后台曝光/点击/涨粉/投放)直接说拿不到并让用户截图,禁止编造对标点赞收藏数或自号数据。不承诺涨粉、爆款、上热门或收益。
