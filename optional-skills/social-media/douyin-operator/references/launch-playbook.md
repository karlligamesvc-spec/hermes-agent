# 抖音起号操作手册

这份参考手册把抖音起号经验提炼成可复用、合规优先的操作流程。发布时间、推荐池、标签权重、DOU+/千川投放、星图报备等说法都按经验假设处理,不视为官方或实时规则。

> **对标系统、爆款转写拆解、选题库、视频实验矩阵不在这份手册里。** 它们是本技能的核心真数据工作流,活在 `SKILL.md` 的第 3 步(对标拆解)、第 3.5 步(爆款转写拆解)、第 4 步(选题库)、第 5 步(9 条视频实验矩阵)——那几节明确指示调用 `social_trending` / `social_search` / `social_posts` / `creator_top_posts` / `social_content` / `social_comments`(`platform=douyin`)拉真实视频数据,并用 `social_download` / `media_transcribe` / `social_batch_submit` 把爆款口播逐字转写再拆。这里只放 prompt-only 的模板和检查表。

## 1. 需求采集表

用户要求起号计划、账号审计、选题库或脚本系统时使用。

| 字段 | 询问或推断 |
| --- | --- |
| 目标 | IP/人设信任、知识科普、剧情娱乐、种草带货、本地门店、引流转化或测试 |
| 赛道 | 类目、细分类目、首月最窄切入口 |
| 观众 | 谁有痛点、好奇心、消费能力或看完动机 |
| 变现 | 账号最终要卖什么、验证什么或沉淀什么(星图/佣金/私域/门店/电商/课程) |
| 证明 | 实拍视频、出镜、案例、资质、经验、对比、用户问题、研究、故事素材 |
| 产能 | 每周视频数、拍摄剪辑能力、出镜/口播能力、封面制作、脚本速度、首轮冲刺周期 |
| 阶段 | 新号、沉寂号、内容不稳定、低播放、限流风险、活跃中或放大期 |
| 约束 | 合规类别(医美/医疗/金融/减肥/教育)、广告法边界、版权(视频/BGM/肖像)、隐私、品牌语气、导流规则 |

## 2. 起号模式判断

| 模式 | 适用情况 | 优先优化 | 注意事项 |
| --- | --- | --- | --- |
| IP/人设信任号 | 用户想做长期品牌、服务、产品或观众信任 | 清晰承诺、人设一致、稳定赛道、专业度、留存、转化路径 | 反馈较慢,避免追逐无关热点 |
| 知识科普/干货号 | 用户有专业输出,靠信息密度立号 | 垂直选题系统、钩子结构、信息密度、可信来源、系列化 | 别为了热点牺牲垂直;来源要可核验 |
| 种草/带货号 | 用户想测种草收益或低成本内容题材 | 细分品类、真实体验、生产系统、原创度、版权安全、复盘循环 | 波动大,不能承诺收益或上热门 |
| 修复/限流号 | 账号有旧内容、低播放或定位漂移 | 诊断、清理、垂直冲刺、信任重建 | 先判断是否有违规导流、搬运或错误受众信号 |
| 放大型账号 | 已经有部分有效选题 | 模式扩展、主题簇、系列化、封面/开头模板复用、转化 | 胜出模式没稳定前不要过度扩张 |

默认优先做 IP/人设信任系统。只有当用户明确接受风险和有限上限时,才设计纯流量/带货实验号。

## 3. 主页地基

先产出定位句:

`我帮助[目标观众],用[方法/证明/内容承诺]解决[具体痛点],让他们获得[理想结果]。`

把定位句转成:

- 头像:清晰、可识别、符合领域气质(人设号优先真人/固定形象)。
- 昵称:一个领域关键词,加一个有记忆点的人设或IP名。
- 简介:一句话写清身份、价值和更新承诺,自然嵌入可搜关键词。
- 主页背景图:强化人设/价值主张,可含一句 slogan。
- 置顶视频方向:代表作/合集/自我介绍,承接主页第一印象。
- 主页引导话术:简介/背景图里的合规下一步动作(如"关注看 XX 系列")。
- 内容支柱:3到5个支柱,分别承接搜索、干货、故事、情绪和转化。
- 关键词宇宙:品类词、痛点词、场景词、人群词、对标邻近词(既做标签也做可搜标题词)。

### 账号准备度检查表

| 模块 | 检查点 |
| --- | --- |
| 主页 | 头像、昵称、简介、背景图、置顶方向、主页引导都指向同一个观众和价值 |
| 信号 | 首轮内容保持垂直,核心关键词自然出现在标题/口播/字幕/标签中 |
| 素材 | 起号前至少准备10到20份源素材:实拍视频、出镜片段、案例、问题、对比、故事 |
| 对标 | 已用 social 工具真实拉过 5-10 个对标账号、20-50 条对标视频(SKILL.md 第 3 步) |
| 爆款拆解 | 已用 media_transcribe 真实转写 3-5 条爆款,填了钩子/结构/结尾拆解表(SKILL.md 第 3.5 步) |
| 封面 | 已定 2-3 套封面/首帧模板方向(大字/出镜叠字/悬念),每条视频出 3 版选 1 |
| 发布 | 有口播脚本简报、9 条实验矩阵、30天内容日历、复盘表和风险检查表 |
| 合规 | 不做刷量、评论自动回复、私信自动化、隐藏联系方式、广告法极限词、盗视频盗 BGM |

## 4. 前 3 秒钩子模式

前 3 秒决定完播起点,用于设计备选,不用于夸大事实。在 SKILL.md 第 3.5 步从真实爆款逐字稿里归纳时,可对照这张表命名模式。

| 模式 | 用法(第一句台词示例) |
| --- | --- |
| 结论前置法 | 把最反直觉的结论甩在最前,例如"这三个习惯,越努力越穷" |
| 痛点直戳法 | 开头直接点名痛点,例如"皮肤黄的人,千万别穿这三种颜色" |
| 数字法 | 让价值具体,例如"记住这 4 个字,退休金多领十万" |
| 反差法 | 制造张力,例如"我以为是智商税,用完真香" |
| 身份点名法 | 圈定人群划走的人回头,例如"打工人/学生党/宝妈,划走前看一眼" |
| 悬念/留扣法 | 留一个具体信息缺口,例如"最后一个方法,90% 的人都做反了" |
| 提问法 | 直接说出观众心里的问题 |
| 避坑法 | "别再踩这些坑""XX 千万别买",制造损失规避 |
| 高赞评论角度 | 从真实评论(`social_comments`)重建角度,不复制原文 |

每条视频先写多个前 3 秒开场白,再按观众意图、具体程度、可信度和广告法风险选;**开场白必须逐字写死**(它是完播的第一道关)。**极限词(最/第一/国家级/永久/根治等)一律不用。**

## 5. 封面 / 首帧模式清单

抖音封面/首帧是第一入口(信息流里决定点不点、留不留)。在 SKILL.md 第 3 步从真实爆款封面归纳时,可对照这张表命名模式;第 6 步出 3 版封面时从中选不同方向。

| 封面模式 | 用法 | 适合 |
| --- | --- | --- |
| 大字纯色底 | 主标题超大字 + 纯色/浅底,信息一眼看清 | 干货、清单、观点、科普 |
| 人物出镜叠字 | 真人出镜(表情/动作)+ 左上/顶部标题字 | 人设号、口播、剧情 |
| 前后对比 | 左右/上下 before-after + 结果字 | 变美、改造、测评 |
| 悬念遮挡 | 关键信息半遮/打码 + 疑问字,勾好奇 | 揭秘、爆料、教程 |
| 实拍产品特写 | 产品清晰特写 + 卖点字 | 种草、开箱 |
| 步骤/清单编号 | 首帧带序号或"第几步" | 教程、合集、避坑 |
| 表情/情绪脸 | 夸张表情占画面 + 一句情绪字 | 剧情、情绪、娱乐 |

**封面文案原则:** 主标题≤两行、大而可读;副信息补一句;不用极限词;不搬对标封面,由生图能力生成 3 版或用户自有素材制作。

## 6. 爆款转写拆解表模板(配合 SKILL.md 第 3.5 步)

对每条要深拆的爆款,先 `media_transcribe`(传分享链接原文,一步下载+转写)拿到逐字稿,再填:

| 模块 | 从逐字稿 + content + comments 抽 |
| --- | --- |
| 视频 | 链接 / 标题 / 作者 / 公开点赞·评论·收藏·转发 |
| 前 3 秒钩子 | 逐字稿开头一句 + 命中的钩子模式(第 4 节) |
| 内容结构 | 逐字稿骨架:清单/教程/故事/观点/对比/揭秘/案例 |
| 信息节奏 | 多少秒一个信息点、有无反转/加速/留扣;口播密度 |
| 情绪触发 | 好奇/身份/焦虑/松弛/省钱/变好/爽感 |
| 结尾钩子 | 逐字稿结尾怎么收(关注/评论/看下集/主页/合拍) |
| 封面/首帧 | 命中的封面模式(第 5 节) |
| 重建方向 | 用用户自己的选题/素材/人设改写,不搬原话/画面/BGM |

**批量拆一个作者或合集**:`social_batch_submit creator_url=<主页或合集分享链接> min_likes=… top=…` → `job_id` → `social_batch_status` 轮询 → 平台出 Word/Excel。异步任务只提交/轮询/发产物,不自己起脚本。

## 7. 9 条视频实验矩阵模板(配合 SKILL.md 第 5 步)

从选题库挑 3 个最强选题,每个配 3 种形式,落 3×3:

| # | 选题 | 形式 | brief(一句话) | 前 3 秒钩子(逐字开场白) | 计划发布时间 | 发布后公开信号(赞/评/藏/转) | 后台精确数据(用户截图:播放/完播/涨粉) |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | A | 口播出镜 |  |  |  |  |  |
| 2 | A | 空镜+字幕 |  |  |  |  |  |
| 3 | A | 剧情/教程 |  |  |  |  |  |
| 4 | B | 口播出镜 |  |  |  |  |  |
| … | … | … |  |  |  |  |  |
| 9 | C | 剧情/教程 |  |  |  |  |  |

- **回采**:每条发布后 `social_posts` + `social_content` 回采公开信号;精确播放/完播/涨粉让用户从创作者中心截图补最后两列。
- **横向比 + 淘汰**:哪个选题/形式/钩子拿到更强正向信号,跑赢的系列化、跑输的换掉。
- **诚实**:样本只有 9 条,**信号 ≠ 定论**;单条爆/扑是弱假设,别当规律,需多轮矩阵累积。

## 8. 口播脚本简报模板

写脚本前先填这张表(SKILL.md 第 6 步的完整字段)。

| 模块 | 内容 |
| --- | --- |
| 视频任务 | 搜索 / 干货 / 故事 / 情绪 / 避坑 / 揭秘 / 转化 |
| 目标观众 | 谁最应该看完这条视频 |
| 观众状态 | 痛点、好奇、误解、恐惧、渴望、反对意见 |
| 关键词 | 1个主关键词,加2到4个相关词 |
| 参考钩子 | 来自第 3.5 步真实爆款拆解的钩子结构(不抄原话) |
| 标题组 | 6到10个标题,并标出推荐标题 |
| 前 3 秒开场白 | **逐字**写死第一句台词(决定完播) |
| 分镜口播脚本 | 逐条:镜头 / 口播逐字 / 屏幕字幕 / 时长 |
| 封面/首帧文案 | 主标题字(≤两行)+ 副信息 + 封面模式方向(出 3 版) |
| 证明材料 | 用户自有实拍、出镜、案例、对比、经验、来源或过程 |
| 风格 | 口语化、短句、字幕短促,前 3 秒给结果或反差 |
| 行动引导 | 与视频任务匹配的一个合规动作(关注/评论/看主页/看下集) |
| 风险检查 | 广告法极限词、版权(视频/BGM/肖像)、隐私、医美/医疗/金融/减肥、导流合规、平台规则 |

### 口语化改写

草稿写完后,把它当作说给朋友听的一段话:

- 把书面腔改成能说出口的短句。
- 字幕短促,一屏一个信息点。
- 加入具体场景、例子和利害关系。
- 删除空泛总结段。
- 一条视频只保留一个核心记忆点。
- 前 3 秒必须给出结果或反差,别铺垫。

## 9. 提示词模板

以下模板只能作为起点,必须按用户赛道、素材、证明和语气大幅定制。**注意:模板里的"对标"步骤应替换为 SKILL.md 第 3 / 3.5 步的真实工具调用(拉真实视频 + 转写拆解),而不是让模型凭空想象对标账号。**

### 口播视频提示词

```text
你正在为[账号定位]起草一条抖音口播视频脚本。

目标观众:[观众]
视频任务:[搜索/干货/故事/情绪/避坑/揭秘/转化]
核心痛点或好奇心:[痛点]
主关键词:[关键词]
参考钩子:[来自真实爆款逐字稿拆解的钩子结构,不抄原话]
用户自有素材:[实拍、出镜、案例、对比、故事、研究资料]
指定形式:[口播出镜/空镜口播/教程]
风险边界:不用广告法极限词,不复制他人口播,不盗视频盗 BGM,不编造数据,不设计评论自动回复/私信自动化/隐藏联系方式。

先输出原创脚本简报:
1. 8个标题选项,并标出最好的3个。
2. 前 3 秒开场白:逐字写死第一句台词(3个备选)。
3. 封面/首帧文案:主标题字(≤两行)+ 副信息 + 3 个封面方向。
4. 分镜口播脚本表(镜头 / 口播逐字 / 屏幕字幕 / 时长)。
5. 每段需要的证明材料(优先用户实拍/出镜)。
6. 合规主页引导 / 关注引导。
7. 广告法、版权(视频/BGM/肖像)、原创度风险自查。

然后用中文写口播全文,要求口语化、短句、前 3 秒给结果或反差、字幕短促。
```

### 剧情/演示视频提示词

```text
为[账号定位]创作一条抖音剧情/教程演示脚本。

目标观众:[观众]
主题:[主题]
时长目标:[15-60 秒]
主关键词与标签:[关键词 + 3-6 标签(来自真实数据)]

输出:
1. 首帧封面文案(标题字≤两行 + 方向,出 3 版)。
2. 前 3 秒钩子(逐字,留住划走的人)。
3. 分镜脚本表(镜头 / 动作或口播 / 屏幕字幕 / 时长)。
4. 结尾行动引导(合规:关注/评论/看主页/看下集)。
5. 广告法、版权(画面/BGM/肖像)自查。

口播口语化,字幕短促,前 3 秒必须给出结果或反差。不用极限词,不搬运他人素材/BGM。
```

### 图文(抖音图文)提示词

```text
为[账号定位]起草一条抖音图文(多图)。

目标观众:[观众]
选题:[选题]
主关键词与标签:[关键词 + 3-6 标签(来自真实数据)]
```

## 10. 首个30天起号日历

| 阶段 | 天数 | 目标 | 工作 |
| --- | --- | --- | --- |
| 地基 | 1-3 | 建立账号信号和观众承诺 | 定位句、主页六件、封面模板方向、3到5个内容支柱 |

| 脚本储备 | 6-12 | 避免临时赶稿 | 准备 9 条实验矩阵的 brief + 前 3 秒开场白 + 封面各出 3 版选 1 |
| 实验冲刺 | 8-22 | 跑 3×3 矩阵测信号 | 在一个赛道内稳定发布 9 条实验,回采公开信号 + 用户补后台截图,横向对比 |
| 模式扩展 | 21-27 | 复用早期有效模式 | 改写跑赢的选题/钩子/形式,做系列,优化标题/封面 |
| 复盘 | 28-30 | 确定下轮重点 | 排名选题、钩子、形式、封面、行动引导、产能瓶颈和风险;淘汰弱选题,出下轮矩阵假设 |

源经验提到日更和特定时段发布,这只能作为实验假设,不是保证。可持续质量优先于强行日更。

## 11. 指标诊断

按失败环节诊断。**精确播放/完播率/5s 完播/平均播放时长/涨粉/转粉率数据需用户从抖音创作者中心/巨量后台截图提供,外部工具采不到;工具只给公开点赞/评论/收藏/转发信号。**

| 现象 | 可能原因 | 修复动作 |
| --- | --- | --- |
| 播放低(不进流量池) | 账号信号不清、赛道漂移、可能限流(导流/搬运/违规)、首轮内容弱、标签/关键词不精准 | 收紧主页,保持垂直,检查违规导流,优化可搜关键词与标签 |
| 有曝光但完播低(前段掉) | 前 3 秒钩子弱、封面/首帧承诺弱、开头铺垫太长 | 逐字重写前 3 秒开场白(结论前置/痛点直戳),重做封面(3 版选 1) |
| 中段掉(完播中段下滑) | 信息密度低、节奏拖、没留扣子 | 提高信息密度,加反转/加速/留扣,删口水段 |
| 看完但点赞低 | 没有"值得赞"的一句、共鸣不足 | 结尾给一句可认同/可收藏的总结,增加价值密度 |
| 看完但评论低 | 没有明确问题,没有身份或决策张力 | 结尾问一个和观众处境有关的具体问题 |
| 互动有但关注低 | 内容有用,但账号/人设承诺不清 | 强化简介、系列承诺、置顶视频和内容一致性 |
| 关注有但转化弱 | 行动引导不清或太早硬广 | 用合规主页引导/官方留资功能/团购挂载,别用私信自动化 |

类别内相对基准比通用阈值更重要。**热点判断用 `social_trending`(抖音公开热门榜)+ `social_search` 近期高互动视频;实验矩阵样本小,单周信号只作弱假设。**

## 12. 风险规则与合规替代

把高风险技巧改成安全做法。

| 高风险诉求 | 处理方式 |
| --- | --- |
| "能不能让朋友帮忙点赞评论冲数据?" | 提醒人为互动会污染受众信号并有限流风险;建议私下收集质量反馈,而不是制造平台行为 |
| "能不能买赞/刷量/刷播放?" | 拒绝;转向前 3 秒钩子、封面、选题、完播和复盘机制优化 |
| "能不能挂工具自动回复评论 / 自动私信关键词群发引流?" | **拒绝(红线,hc-075 线)**;评论自动回复/私信自动化违规且伤号,评论区只做人工回复策略 + 置顶评论钩子,留资改合规主页引导或抖音官方功能 |
| "怎么搬运/洗稿爆款视频还不被发现?" | 拒绝洗稿和规避检测;只抽结构和钩子(可转写拆解),用原创拍摄和自有证明重建 |
| "封面/画面/BGM 能不能直接从对标视频拿?" | 拒绝盗视频/盗 BGM/盗肖像;封面用生图能力生成或合法授权/自有实拍,BGM 用授权曲库 |
| "标题能不能写'全网最便宜/第一/永久有效'?" | 拒绝广告法极限词与功效承诺;改成可核验的具体表达 |
| "低播放要不要注销重来?" | 先诊断违规导流、定位漂移和质量问题;只有有明确战略理由时才重开 |
| "怎么在视频/评论里藏微信/外链导流?" | 拒绝绕审核;改用合规主页引导、官方私信菜单、团购/门店挂载或商业功能 |

建议话术:

- "我不能帮你设计评论自动回复、私信自动化或规避审核的方法,但可以把它改成合规的内容、人工回复策略和主页引导系统。"
- "对标只学钩子和结构,不搬原口播/画面/BGM。我们用你的实拍素材重新做。"
- "标题不用极限词,涉及医美/减肥/金融等敏感内容时,先核验当前抖音规则和广告法。"
